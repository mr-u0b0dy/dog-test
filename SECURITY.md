# Security Policy

## Supported Versions

Security fixes are provided on a best-effort basis for:

| Version | Supported |
| ------- | --------- |
| `main` (latest) | ✅ |
| Older releases | ❌ |

If you are on an older revision, please reproduce on the latest `main` branch first.

## Reporting a Vulnerability

Please do **not** open public GitHub issues for security vulnerabilities.

Use one of these private channels:

1. **GitHub Security Advisories (preferred):**
   - Go to the repository Security tab and use **Report a vulnerability**.

Include as much of the following as possible:

- A clear description of the issue and impact.
- Affected commit/tag/version.
- Reproduction steps or proof of concept.
- Any relevant logs, traces, or screenshots.
- Suggested mitigation (if known).

## Response Expectations

- Initial triage response: within **5 business days**.
- Status updates: at least every **10 business days** while triaging/remediating.
- We will coordinate disclosure timing with the reporter.

## Disclosure

After a fix is available, we may publish:

- A security advisory,
- A changelog entry,
- And remediation guidance.

We appreciate responsible disclosure and will acknowledge reports that materially improve project security.
