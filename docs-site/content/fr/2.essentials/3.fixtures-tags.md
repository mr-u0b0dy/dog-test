# Fixtures et tags

## Fixtures

`DT_TEST_F` ajoute des callbacks d’initialisation et de nettoyage.

```c
static int setup(void) { return 0; }
static int teardown(void) { return 0; }
```

Si `setup()` renvoie une valeur non nulle, le corps du test n’est pas exécuté.

## Tags

```c
DT_TEST_T("uart_smoke", DT_RESET_SOFT, NULL, "uart,smoke", uart_smoke) { ... }
```

Le runner peut filtrer avec `--tag smoke`.
