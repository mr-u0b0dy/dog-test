# Monitor Specs

Un monitor spec décrit comment capturer le signal et quelles attentes vérifier.

## Champs

- `protocol`
- `channels`
- `trigger`
- `timeout_ms`
- `expect`

## Exemple

```json
{
  "protocol": "uart",
  "channels": "ch0=tx,ch1=rx",
  "trigger": "start-on-byte:0x55",
  "timeout_ms": 1200,
  "expect": { "protocol": "uart", "min_items": 1, "contains": "0x55" }
}
```

Les clés inconnues dans `expect` sont rejetées à la validation.
