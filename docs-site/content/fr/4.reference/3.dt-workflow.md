# Workflow dog-test

## Lancer un test unique

```bash [Terminal]
python3 tools/dt_runner.py \
  --firmware build/arm-gcc-debug/examples/uart_loopback/uart_loopback \
  --backend pyocd \
  --serial-port /dev/ttyACM0 \
  --reset-mode soft
```

## Lancer un test-plan

```bash [Terminal]
python3 tools/dt_runner.py --test-plan tests/test_plan_smoke.json --junit-xml results.xml
```

## Options fréquentes

- `--retries`
- `--monitor-spec`
- `--tag` / `--filter`
- `--junit-xml`

Le même flux peut être appelé depuis CTest en CI.
