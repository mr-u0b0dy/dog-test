# Архитектура

dog-test состоит из двух уровней: таргет (C) и хост (Python).

## Компоненты

- `framework/` — регистрация тестов, макросы, `DT_EVENT`
- `tools/dt_runner.py` — orchestration: flash/reset/capture
- `tools/logic/*` — декодеры и проверки monitor spec

## Поток выполнения

1. Сборка firmware
2. Загрузка на устройство
3. Сброс по выбранной стратегии
4. Сбор и парсинг `DT_EVENT`
5. Итог PASS/FAIL и отчёт

Модель остаётся расширяемой для других языков (например, Rust).
