# Rust Roadmap

## Цель

Поддержать Rust-тесты в том же orchestration-потоке, что и C/C++ тесты.

## Ключевой контракт

Rust-бинарники должны выдавать совместимые строки `DT_EVENT`.

## План

1. Добавить backend сборки через `cargo build --target`
2. Генерировать test-plan из метаданных Rust
3. Поддержать reset/monitor атрибуты
4. Запускать смешанные наборы C + Rust
