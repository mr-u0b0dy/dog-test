# Структура проекта

dog-test разделяет код прошивки и хостовую оркестрацию.

## Основные каталоги

- `framework/`: C-макросы, регистрация тестов, `DT_EVENT`
- `tools/`: `dt_runner.py`, backends, логика декодирования
- `examples/`: примеры UART / I2C / SPI
- `tests/`: self-tests и CTest интеграция
- `docs-site/`: документация на Nuxt/Docus

## Типовой цикл

1. Конфигурация CMake
2. Сборка бинарника
3. Запуск `dt_runner.py`
4. Анализ результата и отчёта

Подробно: [Dog Test Workflow](/ru/reference/dt-workflow).
