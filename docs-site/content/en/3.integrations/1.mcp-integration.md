# MCP Integration

The dog-test documentation site includes **Model Context Protocol (MCP)** support via `@nuxtjs/mcp-toolkit`. This allows MCP-compatible AI clients (e.g. GitHub Copilot, Claude) to discover and call tools served by the docs site.

## What is Enabled

The `nuxt.config.ts` configures the MCP toolkit:

```ts [nuxt.config.ts]
modules: ['@nuxtjs/mcp-toolkit'],

mcpToolkit: {
  llm: true,
  docs: true,
},
```

This activates:
- **Auto-discovered MCP tools** from `server/mcp/tools/`
- **Documentation context** for AI assistants
- **Metadata** about available tools and their parameters

## Built-in Tool: `docs-links`

The `docs-links` tool returns documentation URLs for a given locale:

```
Location: server/mcp/tools/docs-links.ts
```

Call it from an MCP client:

```json
{
  "tool": "docs-links",
  "arguments": { "locale": "en" }
}
```

It returns structured links to the key documentation sections, making it easy for AI agents to navigate the docs.

## Adding Custom Tools

Create a new file under `server/mcp/tools/`:

```ts [server/mcp/tools/my-tool.ts]
export default defineMCPTool({
  name: 'my-tool',
  description: 'Description for the AI client',
  parameters: {
    type: 'object',
    properties: {
      query: { type: 'string', description: 'Search query' }
    },
    required: ['query']
  },
  async handler({ query }) {
    // Your logic here
    return { result: `You searched for: ${query}` }
  }
})
```

Tools are auto-discovered — no registration step needed.

## Connecting an MCP Client

1. Start the docs site: `cd docs-site && npm run dev`
2. The MCP endpoint is available at the app's URL.
3. Point your MCP client to the endpoint.
4. The client will discover available tools automatically.
