# Assertions

dog-test provides two families of assertion macros:

- **`HT_ASSERT_*`** — fatal: immediately returns `HT_TEST_FAILED` on failure
- **`HT_EXPECT_*`** — non-fatal: records the failure but continues execution

## Boolean

| Macro | Description |
|-------|-------------|
| `HT_ASSERT_TRUE(expr)` | Fails if `expr` is false/zero |
| `HT_ASSERT_FALSE(expr)` | Fails if `expr` is true/non-zero |

## Integer Comparison

| Macro | Description |
|-------|-------------|
| `HT_ASSERT_EQ(expected, actual)` | `expected == actual` (as `long`) |
| `HT_ASSERT_NE(expected, actual)` | `expected != actual` |
| `HT_ASSERT_GT(a, b)` | `a > b` |
| `HT_ASSERT_GE(a, b)` | `a >= b` |
| `HT_ASSERT_LT(a, b)` | `a < b` |
| `HT_ASSERT_LE(a, b)` | `a <= b` |

## Typed Comparison

| Macro | Description |
|-------|-------------|
| `HT_ASSERT_EQ_U32(expected, actual)` | Unsigned 32-bit equality |
| `HT_ASSERT_EQ_I64(expected, actual)` | Signed 64-bit equality |
| `HT_ASSERT_NE_I64(expected, actual)` | Signed 64-bit inequality |
| `HT_ASSERT_EQ_U64(expected, actual)` | Unsigned 64-bit equality |
| `HT_ASSERT_NE_U64(expected, actual)` | Unsigned 64-bit inequality |

## Float

| Macro | Description |
|-------|-------------|
| `HT_ASSERT_FLOAT_NEAR(expected, actual, epsilon)` | `|expected - actual| <= epsilon` |

On failure, prints the expected value, actual value, and tolerance.

## Bitmask

| Macro | Description |
|-------|-------------|
| `HT_ASSERT_BITS(mask, expected, actual)` | `(expected & mask) == (actual & mask)` |

Useful for checking specific register bits without comparing the entire value.

## Array

| Macro | Description |
|-------|-------------|
| `HT_ASSERT_ARRAY_EQ(expected, actual, count)` | Element-wise equality with index-of-first-difference reporting |

## Strings

| Macro | Description |
|-------|-------------|
| `HT_ASSERT_STR_EQ(expected, actual)` | String equality via `strcmp` |
| `HT_ASSERT_STR_CONTAINS(haystack, needle)` | Substring check via `strstr` |

## Memory

| Macro | Description |
|-------|-------------|
| `HT_ASSERT_MEM_EQ(expected, actual, len)` | Byte-for-byte equality via `memcmp` |

## Pointers

| Macro | Description |
|-------|-------------|
| `HT_ASSERT_NULL(ptr)` | Pointer is `NULL` |
| `HT_ASSERT_NOT_NULL(ptr)` | Pointer is non-`NULL` |

## Range

| Macro | Description |
|-------|-------------|
| `HT_ASSERT_IN_RANGE(val, lo, hi)` | `lo <= val <= hi` |

## Custom Message

| Macro | Description |
|-------|-------------|
| `HT_ASSERT_MSG(expr, message)` | Fails with a custom message string |

## Non-Fatal (HT_EXPECT_*)

Every `HT_ASSERT_*` macro has a corresponding `HT_EXPECT_*` variant that
records the failure without returning from the test function. The runner
promotes the test result to `HT_TEST_FAILED` after it returns if any
expectations failed.

```c
HT_TEST("multi_check", HT_RESET_NONE, NULL, multi_check) {
    HT_EXPECT_EQ(1, get_a());  // continues even if this fails
    HT_EXPECT_EQ(2, get_b());  // this check still runs
    HT_EXPECT_EQ(3, get_c());  // and this one too
    return HT_TEST_PASSED;     // promoted to FAILED if any expectation failed
}
```
