# Writing Tests

Every dog-test test case is a C function that returns a `ht_test_result_t` and
is auto-registered at startup via GCC's `__attribute__((constructor))`.

## Basic Test

```c
#include "hiltest/assert.h"
#include "hiltest/test_case.h"

HT_TEST("my_first_test", HT_RESET_NONE, NULL, my_first_test) {
    HT_ASSERT_EQ(2 + 2, 4);
    return HT_TEST_PASSED;
}

HT_MAIN()
```

The `HT_TEST` macro accepts:
1. **name** — a string identifier for the test
2. **reset mode** — `HT_RESET_NONE`, `HT_RESET_SOFT`, or `HT_RESET_HARD`
3. **monitor spec** — pointer to an `ht_monitor_spec_t`, or `NULL`
4. **function name** — the C function that implements the test

## Adding Tags

Use `HT_TEST_T` to add comma-separated tags for filtering:

```c
HT_TEST_T("uart_smoke", HT_RESET_SOFT, NULL, "uart,smoke", uart_smoke) {
    HT_ASSERT_TRUE(1);
    return HT_TEST_PASSED;
}
```

Run only tagged tests with `--tag smoke` on the host runner.

## Using Fixtures

`HT_TEST_F` adds setup/teardown callbacks:

```c
static int counter = 0;

static int my_setup(void)    { counter = 42; return 0; }
static int my_teardown(void) { counter = 0;  return 0; }

HT_TEST_F("fixture_demo", HT_RESET_NONE, NULL,
           my_setup, my_teardown, fixture_demo) {
    HT_ASSERT_EQ(counter, 42);
    return HT_TEST_PASSED;
}
```

Setup returns `0` on success. A non-zero return aborts the test with a setup
failure. Teardown always runs, even if the test fails.

## Skipping Tests

```c
HT_TEST("conditional", HT_RESET_NONE, NULL, conditional_test) {
    if (!hw_available()) {
        HT_SKIP("hardware not connected");
    }
    // ... real assertions ...
    return HT_TEST_PASSED;
}
```

## Entry Point

Every test binary needs `HT_MAIN()` which provides a `main()` that emits the
boot event, runs all tests, and returns the failure count as the process exit
code.
