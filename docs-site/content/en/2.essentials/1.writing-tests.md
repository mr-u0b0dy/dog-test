# Writing Tests

Every dog-test test case is a C function that returns a `dt_test_result_t` and
is auto-registered at startup via GCC's `__attribute__((constructor))`.

## Basic Test

```c
#include "dogtest/assert.h"
#include "dogtest/test_case.h"

DT_TEST("my_first_test", DT_RESET_NONE, NULL, my_first_test) {
    DT_ASSERT_EQ(2 + 2, 4);
    return DT_TEST_PASSED;
}

DT_MAIN()
```

The `DT_TEST` macro accepts:
1. **name** — a string identifier for the test
2. **reset mode** — `DT_RESET_NONE`, `DT_RESET_SOFT`, or `DT_RESET_HARD`
3. **monitor spec** — pointer to an `dt_monitor_spec_t`, or `NULL`
4. **function name** — the C function that implements the test

## Adding Tags

Use `DT_TEST_T` to add comma-separated tags for filtering:

```c
DT_TEST_T("uart_smoke", DT_RESET_SOFT, NULL, "uart,smoke", uart_smoke) {
    DT_ASSERT_TRUE(1);
    return DT_TEST_PASSED;
}
```

Run only tagged tests with `--tag smoke` on the host runner.

## Using Fixtures

`DT_TEST_F` adds setup/teardown callbacks:

```c
static int counter = 0;

static int my_setup(void)    { counter = 42; return 0; }
static int my_teardown(void) { counter = 0;  return 0; }

DT_TEST_F("fixture_demo", DT_RESET_NONE, NULL,
           my_setup, my_teardown, fixture_demo) {
    DT_ASSERT_EQ(counter, 42);
    return DT_TEST_PASSED;
}
```

Setup returns `0` on success. A non-zero return aborts the test with a setup
failure. Teardown always runs, even if the test fails.

## Skipping Tests

```c
DT_TEST("conditional", DT_RESET_NONE, NULL, conditional_test) {
    if (!hw_available()) {
        DT_SKIP("hardware not connected");
    }
    // ... real assertions ...
    return DT_TEST_PASSED;
}
```

## Entry Point

Every test binary needs `DT_MAIN()` which provides a `main()` that emits the
boot event, runs all tests, and returns the failure count as the process exit
code.
