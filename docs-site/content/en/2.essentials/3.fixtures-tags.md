# Fixtures & Tags

## Fixtures

Fixtures provide per-test setup and teardown. They are ordinary C functions
returning `int` (0 = success, non-zero = abort):

```c
static int setup(void)    { /* allocate resources */ return 0; }
static int teardown(void) { /* free resources */     return 0; }

HT_TEST_F("resource_test", HT_RESET_NONE, NULL,
           setup, teardown, resource_test) {
    // setup() has already run
    HT_ASSERT_NOT_NULL(get_resource());
    return HT_TEST_PASSED;
    // teardown() runs automatically — even on failure
}
```

### Setup Failure

If `setup()` returns non-zero, the test is marked **failed** with reason
`"setup"` and the test function body is not executed. Teardown is **not**
called if setup failed.

### Combining Fixtures with Tags

Use `HT_TEST_FT` for the full combination:

```c
HT_TEST_FT("full_test", HT_RESET_SOFT, NULL,
            setup, teardown, "integration,slow", full_test) {
    // ...
    return HT_TEST_PASSED;
}
```

## Tags

Tags are comma-separated strings attached to test registrations:

```c
HT_TEST_T("fast_check", HT_RESET_NONE, NULL, "unit,fast", fast_check) { ... }
HT_TEST_T("slow_check", HT_RESET_HARD, NULL, "integration,slow", slow_check) { ... }
```

### Filtering by Tag

On the host side, pass `--tag <tag>` to the runner:

```bash
python3 tools/hil_runner.py --firmware build/... --tag fast
```

On the target side, `ht_run_tests_by_tag("fast")` runs only matching tests.

### Name Filtering

Pass `--filter <substring>` to run tests whose name contains the given string:

```bash
python3 tools/hil_runner.py --firmware build/... --filter uart
```
