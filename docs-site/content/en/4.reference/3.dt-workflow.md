# Dog Test Workflow

## Prerequisites

| Requirement | Notes |
|---|---|
| CMake ≥ 3.22 | Build system |
| Ninja | Build backend |
| Python ≥ 3.10 | Host orchestrator |
| pyOCD or OpenOCD | Flash/reset backend |
| pyserial | Serial capture |
| Saleae Logic 2 + `saleae-automation` | Optional — protocol analysis |

## CLI Reference

```
python3 tools/dt_runner.py [OPTIONS]
```

### Core Arguments

| Flag | Description |
|---|---|
| `--firmware PATH` | Path to the firmware binary to flash |
| `--backend {pyocd,openocd}` | Flash/reset backend to use |
| `--serial-port PORT` | Serial port for DT_EVENT capture (e.g. `/dev/ttyACM0`) |
| `--reset-mode {none,soft,hard}` | Reset policy before test execution |
| `--test-plan PATH` | JSON test plan file (runs multiple tests) |

### Test Plan Options

| Flag | Description |
|---|---|
| `--reset-between-tests` | Force a hardware reset between each test in a plan |

### Build Integration

| Flag | Description |
|---|---|
| `--configure-preset NAME` | CMake configure preset to use |
| `--build-preset NAME` | CMake build preset to use |
| `--target NAME` | CMake build target |

### OpenOCD Options

| Flag | Description |
|---|---|
| `--openocd-interface PATH` | OpenOCD interface config (e.g. `interface/cmsis-dap.cfg`) |
| `--openocd-target PATH` | OpenOCD target config (e.g. `target/stm32f4x.cfg`) |

### Output & Reliability

| Flag | Description |
|---|---|
| `--retries N` | Retry failed tests up to N additional times (default: 0) |
| `--junit-xml PATH` | Write JUnit XML report to PATH |
| `-v` / `--verbose` | Enable debug-level logging |
| `-q` / `--quiet` | Suppress all output except errors |

## Single Test Run

Flash and run a single firmware binary:

```bash [Terminal]
python3 tools/dt_runner.py \
  --firmware build/arm-gcc-debug/examples/uart_loopback/uart_loopback \
  --backend pyocd \
  --serial-port /dev/ttyACM0 \
  --reset-mode soft
```

### With OpenOCD backend

```bash [Terminal]
python3 tools/dt_runner.py \
  --firmware build/arm-gcc-debug/examples/uart_loopback/uart_loopback \
  --backend openocd \
  --openocd-interface interface/cmsis-dap.cfg \
  --openocd-target target/stm32f4x.cfg \
  --serial-port /dev/ttyACM0 \
  --reset-mode hard
```

### With retries and JUnit output

```bash [Terminal]
python3 tools/dt_runner.py \
  --firmware build/arm-gcc-debug/examples/uart_loopback/uart_loopback \
  --backend pyocd \
  --serial-port /dev/ttyACM0 \
  --reset-mode soft \
  --retries 2 \
  --junit-xml results.xml \
  -v
```

## Test Plan Run

Run a suite of tests from a JSON plan:

```bash [Terminal]
python3 tools/dt_runner.py \
  --test-plan tests/test_plan_smoke.json \
  --backend pyocd \
  --serial-port /dev/ttyACM0 \
  --reset-between-tests \
  --junit-xml smoke-results.xml
```

### Test plan JSON format

```json [tests/test_plan_smoke.json]
{
  "tests": [
    {
      "name": "uart_loopback_once",
      "firmware": "build/host-debug/examples/uart_loopback/uart_loopback",
      "reset_mode": "none"
    },
    {
      "name": "i2c_smoke_once",
      "firmware": "build/host-debug/examples/i2c_smoke/i2c_smoke",
      "reset_mode": "soft",
      "monitor": {
        "protocol": "uart",
        "channels": "ch0=tx",
        "trigger": "",
        "timeout_ms": 1200,
        "expect": { "protocol": "uart", "min_items": 1 }
      }
    }
  ]
}
```

Each entry can specify:
- `name` — Test display name
- `firmware` — Path to the binary (relative to project root)
- `reset_mode` — `none`, `soft`, or `hard`
- `monitor` — Optional monitor spec object (or path to a JSON monitor spec file) for Saleae verification

## Monitor Spec Format

```json
{
  "protocol": "uart",
  "channels": "ch0=tx,ch1=rx",
  "trigger": "start-on-byte:0x55",
  "timeout_ms": 1200,
  "expect": {
    "protocol": "uart",
    "min_items": 1,
    "contains": "0x55"
  }
}
```

| Field | Description |
|---|---|
| `protocol` | `uart`, `i2c`, or `spi` |
| `channels` | Saleae channel mapping |
| `trigger` | Capture start condition |
| `timeout_ms` | Capture timeout in milliseconds |
| `expect.min_items` | Minimum number of decoded frames expected |
| `expect.contains` | Value that must appear in at least one frame |

## CTest Integration

dog-test provides two CMake helper functions in `cmake/DogTest.cmake`:

### `dt_add_runner_test()`

Register a single firmware binary as a CTest:

```cmake
dt_add_runner_test(uart_loopback
    $<TARGET_FILE:uart_loopback>
    RESET_MODE soft
    TIMEOUT 120
    LABELS dt uart
)
```

### `dt_add_plan_test()`

Register a JSON test plan as a CTest:

```cmake
dt_add_plan_test(smoke_suite
    ${CMAKE_SOURCE_DIR}/tests/test_plan_smoke.json
    TIMEOUT 300
    LABELS dt plan
)
```

Both functions set the working directory to the project root and apply the specified timeout.

Run all registered tests:

```bash [Terminal]
ctest --preset host-debug
```

Or filter by label:

```bash [Terminal]
ctest --preset host-debug -L uart
```
