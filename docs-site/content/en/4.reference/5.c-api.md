# C API Reference

All public API lives in three headers under `framework/include/hiltest/`.

## `test_case.h` — Test Registration & Execution

### Types

#### `ht_reset_mode_t`

```c
typedef enum {
    HT_RESET_NONE = 0,  // No reset before test
    HT_RESET_SOFT = 1,  // Software reset
    HT_RESET_HARD = 2   // Hardware reset (power cycle / NRST)
} ht_reset_mode_t;
```

#### `ht_test_result_t`

```c
typedef enum {
    HT_TEST_PASSED  = 0,
    HT_TEST_FAILED  = 1,
    HT_TEST_SKIPPED = 2
} ht_test_result_t;
```

#### `ht_monitor_spec_t`

```c
typedef struct {
    const char* protocol;    // "uart", "i2c", or "spi"
    const char* channels;    // Channel mapping, e.g. "ch0=tx,ch1=rx"
    const char* trigger;     // Capture trigger condition
    uint32_t    timeout_ms;  // Capture timeout in ms
} ht_monitor_spec_t;
```

#### `ht_test_case_t`

```c
typedef struct {
    const char*              name;      // Test display name
    ht_test_fn_t             run;       // Test function pointer
    ht_reset_mode_t          reset_mode;
    const ht_monitor_spec_t* monitor;   // Optional Saleae monitor spec (NULL = none)
    ht_fixture_fn_t          setup;     // Called before run(); may be NULL
    ht_fixture_fn_t          teardown;  // Called after run();  may be NULL
    const char*              tags;      // Comma-separated tags; may be NULL
} ht_test_case_t;
```

### Registration Macros

#### `HT_TEST(name, reset_mode, monitor_ptr, fn_name)`

Register a basic test. The test function body follows the macro.

```c
HT_TEST("my_test", HT_RESET_NONE, NULL, my_test) {
    HT_ASSERT_TRUE(1);
    return HT_TEST_PASSED;
}
```

| Parameter | Type | Description |
|---|---|---|
| `name` | `const char*` | Display name for the test |
| `reset_mode` | `ht_reset_mode_t` | Reset policy before execution |
| `monitor_ptr` | `const ht_monitor_spec_t*` | Saleae monitor spec, or `NULL` |
| `fn_name` | identifier | C function name (must be unique) |

#### `HT_TEST_T(name, reset_mode, monitor_ptr, tags, fn_name)`

Same as `HT_TEST` but with a `tags` string for filtering.

```c
HT_TEST_T("uart_basic", HT_RESET_SOFT, NULL, "uart,smoke", uart_basic) {
    // ...
    return HT_TEST_PASSED;
}
```

#### `HT_TEST_F(name, reset_mode, monitor_ptr, setup_fn, teardown_fn, fn_name)`

Test with setup/teardown fixtures. Setup and teardown are `int (*)(void)` functions — return 0 for success, non-zero to abort.

```c
static int my_setup(void)    { /* init hardware */ return 0; }
static int my_teardown(void) { /* release hardware */ return 0; }

HT_TEST_F("with_fixture", HT_RESET_HARD, NULL, my_setup, my_teardown, with_fixture) {
    // setup has already run
    HT_ASSERT_TRUE(1);
    return HT_TEST_PASSED;
    // teardown runs after this returns
}
```

#### `HT_TEST_FT(name, reset_mode, monitor_ptr, setup_fn, teardown_fn, tags, fn_name)`

Fixtures + tags combined.

### Entry Point

#### `HT_MAIN()`

Provides a standard `main()` function:

```c
#define HT_MAIN() \
    int main(void) { \
        puts("HT_EVENT boot"); \
        return ht_run_all_tests(); \
    }
```

Place it at the bottom of your test file. The exit code equals the number of failed tests (0 = all passed).

### Runtime Functions

| Function | Returns | Description |
|---|---|---|
| `ht_run_all_tests()` | `int` | Run all registered tests, return failure count |
| `ht_run_tests_filtered(filter)` | `int` | Run tests whose name contains `filter` |
| `ht_run_tests_by_tag(tag)` | `int` | Run tests whose `tags` field contains `tag` |
| `ht_registered_test_count()` | `size_t` | Number of registered tests |
| `ht_registered_test_at(index)` | `const ht_test_case_t*` | Get test by index |

## `assert.h` — Assertions & Expectations

### Fatal Assertions (`HT_ASSERT_*`)

These abort the test on failure by returning `HT_TEST_FAILED`.

| Macro | Description |
|---|---|
| `HT_ASSERT_TRUE(expr)` | Fails if `expr` is zero |
| `HT_ASSERT_FALSE(expr)` | Fails if `expr` is non-zero |
| `HT_ASSERT_EQ(expected, actual)` | Fails if values differ (cast to `long`) |
| `HT_ASSERT_NE(expected, actual)` | Fails if values are equal |
| `HT_ASSERT_GT(a, b)` | Fails if `a <= b` |
| `HT_ASSERT_GE(a, b)` | Fails if `a < b` |
| `HT_ASSERT_LT(a, b)` | Fails if `a >= b` |
| `HT_ASSERT_LE(a, b)` | Fails if `a > b` |
| `HT_ASSERT_EQ_U32(expected, actual)` | Unsigned 32-bit comparison |
| `HT_ASSERT_FLOAT_NEAR(expected, actual, epsilon)` | Fails if `|expected - actual| > epsilon` |
| `HT_ASSERT_IN_RANGE(val, lo, hi)` | Fails if `val < lo` or `val > hi` |
| `HT_ASSERT_NULL(ptr)` | Fails if `ptr != NULL` |
| `HT_ASSERT_NOT_NULL(ptr)` | Fails if `ptr == NULL` |
| `HT_ASSERT_STR_EQ(expected, actual)` | `strcmp`-based string equality |
| `HT_ASSERT_STR_CONTAINS(haystack, needle)` | `strstr`-based substring check |
| `HT_ASSERT_MEM_EQ(expected, actual, len)` | `memcmp`-based byte comparison |
| `HT_ASSERT_MSG(expr, message)` | Custom failure message |

### Non-Fatal Expectations (`HT_EXPECT_*`)

These record a failure but **do not** stop the test. Execution continues, and the test is marked failed at the end if any expectation failed.

| Macro | Description |
|---|---|
| `HT_EXPECT_TRUE(expr)` | Expects `expr` is non-zero |
| `HT_EXPECT_FALSE(expr)` | Expects `expr` is zero |
| `HT_EXPECT_EQ(expected, actual)` | Expects values are equal |
| `HT_EXPECT_NE(expected, actual)` | Expects values differ |
| `HT_EXPECT_GT(a, b)` | Expects `a > b` |
| `HT_EXPECT_NULL(ptr)` | Expects `ptr == NULL` |
| `HT_EXPECT_NOT_NULL(ptr)` | Expects `ptr != NULL` |
| `HT_EXPECT_STR_EQ(expected, actual)` | Expects strings are equal |
| `HT_EXPECT_MSG(expr, message)` | Custom failure message |

### Skip

#### `HT_SKIP(reason)`

Skip the current test with a reason string. The test is marked as skipped (not failed).

```c
HT_TEST("conditional_test", HT_RESET_NONE, NULL, conditional_test) {
    if (!hardware_available()) {
        HT_SKIP("hardware not connected");
    }
    // ...
    return HT_TEST_PASSED;
}
```

## `monitor_spec.h` — Monitor Spec Helpers

Convenience macros for creating `ht_monitor_spec_t` compound literals inline:

| Macro | Protocol |
|---|---|
| `HT_MONITOR_UART(channels, trigger, timeout_ms)` | UART |
| `HT_MONITOR_I2C(channels, trigger, timeout_ms)` | I2C |
| `HT_MONITOR_SPI(channels, trigger, timeout_ms)` | SPI |

Example:

```c
HT_TEST("with_monitor", HT_RESET_SOFT,
        HT_MONITOR_UART("ch0=tx,ch1=rx", "start-on-byte:0x55", 1000),
        with_monitor) {
    // ...
    return HT_TEST_PASSED;
}
```
