# C API Reference

All public API lives in three headers under `framework/include/dogtest/`.

## `test_case.h` — Test Registration & Execution

### Types

#### `dt_reset_mode_t`

```c
typedef enum {
    DT_RESET_NONE = 0,  // No reset before test
    DT_RESET_SOFT = 1,  // Software reset
    DT_RESET_HARD = 2   // Hardware reset (power cycle / NRST)
} dt_reset_mode_t;
```

#### `dt_test_result_t`

```c
typedef enum {
    DT_TEST_PASSED  = 0,
    DT_TEST_FAILED  = 1,
    DT_TEST_SKIPPED = 2
} dt_test_result_t;
```

#### `dt_monitor_spec_t`

```c
typedef struct {
    const char* protocol;    // "uart", "i2c", or "spi"
    const char* channels;    // Channel mapping, e.g. "ch0=tx,ch1=rx"
    const char* trigger;     // Capture trigger condition
    uint32_t    timeout_ms;  // Capture timeout in ms
} dt_monitor_spec_t;
```

#### `dt_test_case_t`

```c
typedef struct {
    const char*              name;      // Test display name
    dt_test_fn_t             run;       // Test function pointer
    dt_reset_mode_t          reset_mode;
    const dt_monitor_spec_t* monitor;   // Optional Saleae monitor spec (NULL = none)
    dt_fixture_fn_t          setup;     // Called before run(); may be NULL
    dt_fixture_fn_t          teardown;  // Called after run();  may be NULL
    const char*              tags;      // Comma-separated tags; may be NULL
} dt_test_case_t;
```

### Registration Macros

#### `DT_TEST(name, reset_mode, monitor_ptr, fn_name)`

Register a basic test. The test function body follows the macro.

```c
DT_TEST("my_test", DT_RESET_NONE, NULL, my_test) {
    DT_ASSERT_TRUE(1);
    return DT_TEST_PASSED;
}
```

| Parameter | Type | Description |
|---|---|---|
| `name` | `const char*` | Display name for the test |
| `reset_mode` | `dt_reset_mode_t` | Reset policy before execution |
| `monitor_ptr` | `const dt_monitor_spec_t*` | Saleae monitor spec, or `NULL` |
| `fn_name` | identifier | C function name (must be unique) |

#### `DT_TEST_T(name, reset_mode, monitor_ptr, tags, fn_name)`

Same as `DT_TEST` but with a `tags` string for filtering.

```c
DT_TEST_T("uart_basic", DT_RESET_SOFT, NULL, "uart,smoke", uart_basic) {
    // ...
    return DT_TEST_PASSED;
}
```

#### `DT_TEST_F(name, reset_mode, monitor_ptr, setup_fn, teardown_fn, fn_name)`

Test with setup/teardown fixtures. Setup and teardown are `int (*)(void)` functions — return 0 for success, non-zero to abort.

```c
static int my_setup(void)    { /* init hardware */ return 0; }
static int my_teardown(void) { /* release hardware */ return 0; }

DT_TEST_F("with_fixture", DT_RESET_HARD, NULL, my_setup, my_teardown, with_fixture) {
    // setup has already run
    DT_ASSERT_TRUE(1);
    return DT_TEST_PASSED;
    // teardown runs after this returns
}
```

#### `DT_TEST_FT(name, reset_mode, monitor_ptr, setup_fn, teardown_fn, tags, fn_name)`

Fixtures + tags combined.

### Entry Point

#### `DT_MAIN()`

Provides a standard `main()` function:

```c
#define DT_MAIN() \
    int main(void) { \
        puts("DT_EVENT boot"); \
        return dt_run_all_tests(); \
    }
```

Place it at the bottom of your test file. The exit code equals the number of failed tests (0 = all passed).

### Runtime Functions

| Function | Returns | Description |
|---|---|---|
| `dt_run_all_tests()` | `int` | Run all registered tests, return failure count |
| `dt_run_tests_filtered(filter)` | `int` | Run tests whose name contains `filter` |
| `dt_run_tests_by_tag(tag)` | `int` | Run tests whose `tags` field contains `tag` |
| `dt_registered_test_count()` | `size_t` | Number of registered tests |
| `dt_registered_test_at(index)` | `const dt_test_case_t*` | Get test by index |
| `dt_reset_registry()` | `void` | Clear all test registrations (useful for testing) |

## `assert.h` — Assertions & Expectations

### Fatal Assertions (`DT_ASSERT_*`)

These abort the test on failure by returning `DT_TEST_FAILED`.

| Macro | Description |
|---|---|
| `DT_ASSERT_TRUE(expr)` | Fails if `expr` is zero |
| `DT_ASSERT_FALSE(expr)` | Fails if `expr` is non-zero |
| `DT_ASSERT_EQ(expected, actual)` | Fails if values differ (cast to `long`) |
| `DT_ASSERT_NE(expected, actual)` | Fails if values are equal |
| `DT_ASSERT_GT(a, b)` | Fails if `a <= b` |
| `DT_ASSERT_GE(a, b)` | Fails if `a < b` |
| `DT_ASSERT_LT(a, b)` | Fails if `a >= b` |
| `DT_ASSERT_LE(a, b)` | Fails if `a > b` |
| `DT_ASSERT_EQ_U32(expected, actual)` | Unsigned 32-bit comparison |
| `DT_ASSERT_FLOAT_NEAR(expected, actual, epsilon)` | Fails if `|expected - actual| > epsilon` |
| `DT_ASSERT_IN_RANGE(val, lo, hi)` | Fails if `val < lo` or `val > hi` |
| `DT_ASSERT_NULL(ptr)` | Fails if `ptr != NULL` |
| `DT_ASSERT_NOT_NULL(ptr)` | Fails if `ptr == NULL` |
| `DT_ASSERT_STR_EQ(expected, actual)` | `strcmp`-based string equality |
| `DT_ASSERT_STR_CONTAINS(haystack, needle)` | `strstr`-based substring check |
| `DT_ASSERT_MEM_EQ(expected, actual, len)` | `memcmp`-based byte comparison |
| `DT_ASSERT_MSG(expr, message)` | Custom failure message |

### Non-Fatal Expectations (`DT_EXPECT_*`)

These record a failure but **do not** stop the test. Execution continues, and the test is marked failed at the end if any expectation failed.

| Macro | Description |
|---|---|
| `DT_EXPECT_TRUE(expr)` | Expects `expr` is non-zero |
| `DT_EXPECT_FALSE(expr)` | Expects `expr` is zero |
| `DT_EXPECT_EQ(expected, actual)` | Expects values are equal |
| `DT_EXPECT_NE(expected, actual)` | Expects values differ |
| `DT_EXPECT_GT(a, b)` | Expects `a > b` |
| `DT_EXPECT_GE(a, b)` | Expects `a >= b` |
| `DT_EXPECT_LT(a, b)` | Expects `a < b` |
| `DT_EXPECT_LE(a, b)` | Expects `a <= b` |
| `DT_EXPECT_EQ_U32(expected, actual)` | Unsigned 32-bit equality |
| `DT_EXPECT_FLOAT_NEAR(expected, actual, epsilon)` | Expects `\|expected - actual\| <= epsilon` |
| `DT_EXPECT_IN_RANGE(val, lo, hi)` | Expects `lo <= val <= hi` |
| `DT_EXPECT_MEM_EQ(expected, actual, len)` | `memcmp`-based byte comparison |
| `DT_EXPECT_STR_CONTAINS(haystack, needle)` | `strstr`-based substring check |
| `DT_EXPECT_NULL(ptr)` | Expects `ptr == NULL` |
| `DT_EXPECT_NOT_NULL(ptr)` | Expects `ptr != NULL` |
| `DT_EXPECT_STR_EQ(expected, actual)` | Expects strings are equal |
| `DT_EXPECT_MSG(expr, message)` | Custom failure message |

### Skip

#### `DT_SKIP(reason)`

Skip the current test with a reason string. The test is marked as skipped (not failed).

```c
DT_TEST("conditional_test", DT_RESET_NONE, NULL, conditional_test) {
    if (!hardware_available()) {
        DT_SKIP("hardware not connected");
    }
    // ...
    return DT_TEST_PASSED;
}
```

## `monitor_spec.h` — Monitor Spec Helpers

Convenience macros for creating `dt_monitor_spec_t` compound literals inline:

| Macro | Protocol |
|---|---|
| `DT_MONITOR_UART(channels, trigger, timeout_ms)` | UART |
| `DT_MONITOR_I2C(channels, trigger, timeout_ms)` | I2C |
| `DT_MONITOR_SPI(channels, trigger, timeout_ms)` | SPI |

Example:

```c
DT_TEST("with_monitor", DT_RESET_SOFT,
        DT_MONITOR_UART("ch0=tx,ch1=rx", "start-on-byte:0x55", 1000),
        with_monitor) {
    // ...
    return DT_TEST_PASSED;
}
```
