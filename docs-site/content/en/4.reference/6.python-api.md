# Python API Reference

The host-side tooling lives in the `tools/` package. Install via:

```bash
pip install -e ".[all]"   # all optional dependencies
pip install -e "."         # core only (pyserial)
```

## CLI — `hil-runner`

After installation, the runner is available as `hil-runner`:

```bash
hil-runner --firmware build/examples/uart_loopback/uart_loopback \
  --skip-flash --skip-reset --skip-monitor --skip-target-exec
```

### Key Options

| Flag | Description |
|------|-------------|
| `--firmware PATH` | Path to the firmware binary |
| `--test-plan PATH` | JSON test plan file |
| `--backend pyocd\|openocd` | Flash/reset backend (default: `pyocd`) |
| `--serial-port PORT` | Serial port for target output |
| `--baudrate N` | Serial baud rate (default: 115200) |
| `--reset-mode none\|soft\|hard` | Reset strategy |
| `--tag TAG` | Run only tests with this tag |
| `--filter STRING` | Run only tests whose name contains this string |
| `--monitor-spec JSON` | Inline JSON monitor specification |
| `--retries N` | Retry count for flaky tests |
| `--junit-xml PATH` | Write JUnit XML results |
| `--dry-run` | Print execution plan without running |
| `--skip-flash` | Skip firmware flashing |
| `--skip-reset` | Skip board reset |
| `--skip-monitor` | Skip logic analyser capture |
| `--skip-target-exec` | Skip serial reading (CI smoke mode) |
| `-v` / `--verbose` | Debug logging |
| `-q` / `--quiet` | Suppress info logging |

### Environment Variables

| Variable | Description |
|----------|-------------|
| `HIL_BOARD_ID` | Default board ID for pyOCD |
| `HIL_SERIAL_PORT` | Default serial port |
| `HIL_BAUDRATE` | Default baud rate |
| `HIL_OPENOCD_INTERFACE` | OpenOCD interface config |
| `HIL_OPENOCD_TARGET` | OpenOCD target config |

## `tools.specs.monitor_contract.MonitorRequest`

Dataclass representing a monitor specification.

```python
@dataclass
class MonitorRequest:
    protocol: str          # "uart", "i2c", or "spi"
    channels: str          # e.g. "ch0=tx,ch1=rx"
    trigger: str           # e.g. "start-on-byte:0x55"
    timeout_ms: int        # capture duration
    expect: dict[str, Any] # expectation schema
```

### `MonitorRequest.from_dict(data)`

Construct from a dict (e.g., parsed JSON). Validates the `expect` schema —
unknown keys raise `ValueError`.

### `MonitorRequest.validate()`

Validates `expect` keys against the allowed set: `protocol`, `min_items`,
`contains`.

## Backend System

### Flash Backends

All flash backends implement `tools.backends.flash.base.FlashBackend`:

```python
class FlashBackend(ABC):
    @abstractmethod
    def flash(self, firmware_path: Path) -> None: ...
```

**Built-in backends:**
- `PyOcdFlashBackend(board_id)` — uses `pyocd flash`
- `OpenOcdFlashBackend(interface_cfg, target_cfg)` — uses `openocd -c program`

### Reset Backends

All reset backends implement `tools.backends.reset.base.ResetBackend`:

```python
class ResetBackend(ABC):
    @abstractmethod
    def soft_reset(self) -> None: ...
    @abstractmethod
    def hard_reset(self) -> None: ...
```

**Built-in backends:**
- `PyOcdResetBackend(board_id)` — uses `pyocd cmd -c reset`
- `OpenOcdResetBackend(interface_cfg, target_cfg)` — uses `openocd -c init; ...`

### Custom Backends

Register a custom backend without modifying `hil_runner.py`:

```python
from tools.backends.flash import register_flash_backend
from tools.backends.flash.base import FlashBackend

class JLinkFlash(FlashBackend):
    def __init__(self, board_id):
        self.board_id = board_id

    def flash(self, firmware_path):
        subprocess.run(["JLinkExe", "-CommandFile", ...], check=True)

register_flash_backend("jlink", JLinkFlash)
```

## Decoder API

### `tools.logic.decoders`

| Function | Description |
|----------|-------------|
| `decode_uart(capture)` | Decode UART frames from Saleae CSV export |
| `decode_i2c(capture)` | Decode I2C transactions |
| `decode_spi(capture)` | Decode SPI transactions |
| `decode_by_protocol(capture)` | Auto-dispatch by `capture.protocol` |

Returns a dict with either `"frames"` (UART) or `"transactions"` (I2C/SPI)
plus `"meta"` with raw capture data.

### `tools.logic.assertions`

```python
evaluate_monitor_expectations(capture: LogicCapture, expect: dict) -> None
```

Raises `AssertionError` if any expectation fails.

### `tools.logic.saleae_adapter`

```python
class SaleaeLogicAdapter:
    @classmethod
    def from_environment(cls) -> SaleaeLogicAdapter:
        """Connect using SALEAE_HOST/SALEAE_PORT env vars."""

    def capture_protocol(self, protocol, channels, trigger, duration_ms) -> LogicCapture:
        """Run a timed capture with the specified analyzer."""
```

## HT_EVENT Protocol

The target emits structured lines that the host parses:

| Event | Format |
|-------|--------|
| Boot | `HT_EVENT boot` |
| Start | `HT_EVENT start name=<name> reset=<N> monitor=<protocol> [tags=<tags>]` |
| Pass | `HT_EVENT pass name=<name>` |
| Fail | `HT_EVENT fail name=<name> reason=<reason> file=<file> line=<N> ...` |
| Skip | `HT_EVENT skip name=<name> reason=<reason>` |
| Summary | `HT_EVENT summary total=<N> passed=<N> failed=<N> skipped=<N>` |

Values containing spaces or `=` are automatically quoted.
