Welcome to **dog-test**, an open-source Hardware-in-the-Loop (HIL) testing framework for embedded C/C++ systems.

## What is dog-test?

dog-test lets you write deterministic integration tests that run on **real hardware** — not simulators. It combines three things:

1. **A lightweight C assertion library** compiled into your firmware.
2. **A Python host orchestrator** that flashes, resets, captures serial output, and evaluates results.
3. **Optional protocol analysis** via Saleae Logic 2 for UART, I2C, and SPI verification.

Tests are described in JSON *test plans* and executed by the Python runner. Results are streamed as structured `HT_EVENT` lines over serial, parsed on the host, and reported as pass/fail — with optional JUnit XML for CI integration.

## Why HIL Testing?

Unit tests and emulators cover logic, but many embedded bugs only surface on real silicon:

- Peripheral clock configurations
- DMA/interrupt timing
- Voltage-level or pull-up issues on I2C/SPI buses
- UART baud-rate mismatches

dog-test makes it practical to catch these issues **automatically**, as part of your CI pipeline or local development workflow.

## Architecture at a Glance

```
┌──────────────────────────────────────┐
│  Target MCU (ARM Cortex-M)           │
│  ┌──────────────┐                    │
│  │ Your firmware │◄─ HT_ASSERT_*     │
│  │ + hiltest lib │── HT_EVENT lines ─┼──► UART
│  └──────────────┘                    │
└──────────────────────────────────────┘
         ▲ flash/reset                 │ serial
         │ (pyOCD / OpenOCD)           ▼
┌──────────────────────────────────────┐
│  Host PC                             │
│  hil_runner.py                       │
│  ├─ flash backend                    │
│  ├─ serial capture & HT_EVENT parse  │
│  ├─ (optional) Saleae Logic capture  │
│  └─ result evaluation & JUnit XML    │
└──────────────────────────────────────┘
```

## Key Features

- **Rich C assertions** — `HT_ASSERT_EQ`, `HT_ASSERT_STR_EQ`, `HT_ASSERT_FLOAT_NEAR`, `HT_ASSERT_MEM_EQ`, plus non-fatal `HT_EXPECT_*` variants.
- **Test tags & filtering** — Tag tests with `HT_TEST_T("uart,smoke", ...)` and run subsets by tag.
- **Fixture support** — `HT_TEST_F` and `HT_TEST_FT` for setup/teardown per test.
- **Multiple flash/reset backends** — pyOCD and OpenOCD, extensible to J-Link or custom probes.
- **Saleae Logic 2 integration** — Capture and decode UART, I2C, SPI frames; assert on protocol behaviour.
- **Monitor specs** — JSON contracts for expected signal behaviour, evaluated automatically.
- **JUnit XML output** — `--junit-xml report.xml` for direct CI integration.
- **Retry support** — `--retries N` to handle flaky hardware.
 - **CMake integration** — `hil_add_runner_test()` and `hil_add_plan_test()` register tests with CTest.
- **Host-side testing** — Compile and run the same assertion library on your development machine.
