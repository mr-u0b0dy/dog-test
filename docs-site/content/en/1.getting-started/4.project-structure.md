### Key headers

| Header | Purpose |
|---|---|
| `test_case.h` | Test registration macros, fixture macros, tag macros, `HT_MAIN()` |
| `assert.h` | Fatal assertions (`HT_ASSERT_*`), non-fatal expectations (`HT_EXPECT_*`), skip |
| `monitor_spec.h` | Emit JSON monitor-spec blocks for Saleae verification |

### `hil_runner.py` CLI

The runner reads a JSON test plan, then for each test entry:

1. Flashes the firmware onto the target.
2. Resets the MCU.
3. Opens a serial port and captures `HT_EVENT` lines.
4. (Optionally) starts a Saleae Logic capture.
5. Parses results, evaluates monitor specs, and reports pass/fail.

See [HIL Workflow](/en/reference/hil-workflow) for full CLI usage.

## `cmake/` вЂ” Build Helpers

```bash
cmake/
в”њв”Ђв”Ђ HilTest.cmake            # hil_add_firmware_test(), hil_add_plan_test()
в””в”Ђв”Ђ toolchains/
    в””в”Ђв”Ђ arm-none-eabi.cmake  # ARM cross-compilation toolchain file
```

`HilTest.cmake` is included automatically by the root `CMakeLists.txt` and provides two functions:

- **`hil_add_firmware_test(NAME ... FIRMWARE ... PORT ... BACKEND ...)`** вЂ” registers a single firmware binary as a CTest.
- **`hil_add_plan_test(NAME ... PLAN ... [TIMEOUT ...])`** вЂ” registers a JSON test plan as a CTest.
