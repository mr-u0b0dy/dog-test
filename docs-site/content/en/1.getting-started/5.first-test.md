This guide walks you through writing a minimal embedded test, building it, and running it on the host (no hardware required for the first steps).

## 1. Create the test source

Create a new directory under `examples/` and add a `main.c`:

```c [examples/my_first_test/main.c]
#include "hiltest/test_case.h"
#include "hiltest/assert.h"

HT_TEST("addition_works", HT_RESET_NONE, NULL, addition_works)
{
    int result = 2 + 2;
    HT_ASSERT_EQ(result, 4);
    return HT_TEST_PASSED;
}

HT_TEST("string_check", HT_RESET_NONE, NULL, string_check)
{
    const char *greeting = "hello";
    HT_ASSERT_STR_EQ(greeting, "hello");
    return HT_TEST_PASSED;
}

HT_MAIN()
```

### What's happening?

- `HT_TEST(name, reset_mode, monitor_ptr, fn_name)` registers a test function. The test body must return `HT_TEST_PASSED`, `HT_TEST_FAILED`, or `HT_TEST_SKIPPED`.
- `HT_ASSERT_EQ` is a fatal assertion — if it fails, the test stops immediately.
- `HT_MAIN()` expands to a `main()` that emits `HT_EVENT boot` and calls `ht_run_all_tests()`.

## 2. Add a CMakeLists.txt

```cmake [examples/my_first_test/CMakeLists.txt]
add_executable(my_first_test main.c)
target_link_libraries(my_first_test PRIVATE hiltest_core)
```

Then register the test with CTest using the helper from `cmake/HilTest.cmake` (e.g. in `tests/CMakeLists.txt`):

```cmake
hil_add_runner_test(my_first_test
    $<TARGET_FILE:my_first_test>
    RESET_MODE none
    LABELS example
    SKIP_FLASH SKIP_RESET SKIP_MONITOR SKIP_TARGET_EXEC
)
```

And include the example directory from the parent `CMakeLists.txt`:

```cmake
add_subdirectory(examples/my_first_test)
```

## 3. Build and run on host

Since the test is pure C with no hardware dependencies, you can build and run it directly on the host:

```bash [Terminal]
cmake --preset host-debug
cmake --build --preset host-debug
./build/host-debug/examples/my_first_test/my_first_test
```

Expected output:

```
HT_EVENT boot
HT_EVENT start name=addition_works reset=0 monitor=none
HT_EVENT pass name=addition_works
HT_EVENT start name=string_check reset=0 monitor=none
HT_EVENT pass name=string_check
HT_EVENT summary total=2 passed=2 failed=0 skipped=0
```

## 4. Add non-fatal expectations

Sometimes you want a test to **continue** after a check fails, to report all issues at once. Use `HT_EXPECT_*` macros:

```c
HT_TEST("multiple_checks", HT_RESET_NONE, NULL, multiple_checks)
{
    int values[] = {1, 2, 3, 4};
    HT_EXPECT_EQ(values[0], 1);
    HT_EXPECT_EQ(values[1], 2);
    HT_EXPECT_EQ(values[2], 3);
    HT_EXPECT_EQ(values[3], 4);
    // If any EXPECT fails, the test continues but is marked as failed at the end.
    return HT_TEST_PASSED;
}
```

## 5. Add tags for filtering

Tag your tests to run subsets:

```c
HT_TEST_T("uart_loopback", HT_RESET_SOFT, NULL, "uart,smoke", uart_loopback)
{
    // This test can be selected with --tag uart or --tag smoke
    HT_ASSERT_TRUE(1);
    return HT_TEST_PASSED;
}
```

## 6. Use fixtures for setup/teardown

```c
static int my_setup(void) {
    // Initialise hardware or state; return 0 on success.
    return 0;
}

static int my_teardown(void) {
    // Cleanup; return 0 on success.
    return 0;
}

HT_TEST_F("buffer_starts_empty", HT_RESET_NONE, NULL,
           my_setup, my_teardown, buffer_starts_empty)
{
    HT_ASSERT_TRUE(1);  // setup has already run
    return HT_TEST_PASSED;
    // teardown runs after this returns
}
```
