## Prerequisites

| Tool | Version | Purpose |
|---|---|---|
| **CMake** | ≥ 3.22 | Build system generator |
| **Ninja** | any | Fast build backend |
| **GCC** or **Clang** | C11-capable | Host compiler for tests |
| **arm-none-eabi-gcc** | any | Cross compiler (for ARM targets) |
| **Python** | ≥ 3.10 | Host orchestrator |
| **pyOCD** or **OpenOCD** | any | Flash & reset backend |
| **pyserial** | any | Serial capture |

Optional: **Saleae Logic 2** + `saleae-automation` Python package for protocol analysis.

## Setup

::steps
### Clone the repository

```bash [Terminal]
git clone https://github.com/mr-u0b0dy/dog-test.git
cd dog-test
```

### Install Python dependencies

```bash [Terminal]
pip install -r requirements.txt
```

This installs `pyserial`, `pyocd`, and `pytest` (for running the framework's own tests).

### Configure the CMake project

dog-test ships with CMake presets. For **host-side** development and testing:

```bash [Terminal]
cmake --preset host-debug
```

For **ARM cross-compilation** (requires `arm-none-eabi-gcc` on `PATH`):

```bash [Terminal]
cmake --preset arm-gcc-debug
```

### Build

```bash [Terminal]
cmake --build --preset host-debug
```

### Run the framework's own tests

```bash [Terminal]
ctest --preset host-debug
```

You should see all tests pass, including the C assertion tests and Python unit tests.
::

## Available CMake Presets

| Preset | Toolchain | Type | Notes |
|---|---|---|---|
| `host-debug` | System GCC/Clang | Debug | Default for development |
| `host-release` | System GCC/Clang | Release | Optimised, for benchmarks |
| `arm-gcc-debug` | arm-none-eabi-gcc | Debug | Cross-compile for targets |
| `arm-gcc-release` | arm-none-eabi-gcc | Release | Cross-compile, optimised |

## Docs Site (optional)

The documentation site lives in `docs-site/` and is powered by Docus (Nuxt):

```bash [Terminal]
cd docs-site
npm install
npm run dev
```

A local preview will be available at <http://localhost:3000>.
