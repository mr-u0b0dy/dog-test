### Key headers

| Header | Purpose |
|---|---|
| `test_case.h` | Test registration macros, fixture macros, tag macros, `DT_MAIN()` |
| `assert.h` | Fatal assertions (`DT_ASSERT_*`), non-fatal expectations (`DT_EXPECT_*`), skip |
| `monitor_spec.h` | Emit JSON monitor-spec blocks for Saleae verification |

### `dt_runner.py` CLI

The runner reads a JSON test plan, then for each test entry:

1. Flashes the firmware onto the target.
2. Resets the MCU.
3. Opens a serial port and captures `DT_EVENT` lines.
4. (Optionally) starts a Saleae Logic capture.
5. Parses results, evaluates monitor specs, and reports pass/fail.

See [Dog Test Workflow](/en/reference/dt-workflow) for full CLI usage.

## `cmake/` вЂ” Build Helpers

```bash
cmake/
в”њв”Ђв”Ђ DogTest.cmake            # dt_add_runner_test(), dt_add_plan_test()
в””в”Ђв”Ђ toolchains/
    в””в”Ђв”Ђ arm-none-eabi.cmake  # ARM cross-compilation toolchain file
```

`DogTest.cmake` is included by `tests/CMakeLists.txt` and provides two functions:

- **`dt_add_runner_test(name firmware [RESET_MODE ...] [TIMEOUT ...] [LABELS ...] [SKIP_*])`** вЂ” registers a single firmware binary as a CTest.
- **`dt_add_plan_test(name plan_json [TIMEOUT ...] [LABELS ...] [RESET_BETWEEN_TESTS] [SKIP_*])`** вЂ” registers a JSON test plan as a CTest.
