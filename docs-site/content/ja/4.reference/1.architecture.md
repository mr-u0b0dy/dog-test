# アーキテクチャ

dog-test はターゲット側 C ライブラリとホスト側 Python ランナーで構成されます。

## 主な層

- `framework/`: テスト登録・アサーション・DT_EVENT 出力
- `tools/dt_runner.py`: フラッシュ、リセット、シリアル収集
- `tools/logic/*`: Saleae 連携、デコード、期待値検証

## 実行フロー

1. ファームウェアをビルド
2. バックエンドで書き込み
3. 必要なリセットを実施
4. `DT_EVENT` を収集して判定

この分離により、言語拡張（例: Rust）にも対応しやすくなります。
