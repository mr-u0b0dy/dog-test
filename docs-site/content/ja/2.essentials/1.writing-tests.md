# テストの書き方

dog-test では `DT_TEST` 系マクロでテストを登録します。

## 基本形

```c
DT_TEST("my_test", DT_RESET_NONE, NULL, my_test) {
  DT_ASSERT_TRUE(1);
  return DT_TEST_PASSED;
}
```

## 代表的な登録マクロ

- `DT_TEST`: 基本テスト
- `DT_TEST_T`: タグ付きテスト
- `DT_TEST_F`: setup/teardown 付き
- `DT_TEST_FT`: fixture + タグ

`DT_MAIN()` を最後に置くことで実行エントリが生成されます。
