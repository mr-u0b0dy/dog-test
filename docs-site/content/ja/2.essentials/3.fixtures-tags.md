# Fixture とタグ

## Fixture

`DT_TEST_F` では、テストの前後で初期化と後始末を実行できます。

```c
static int setup(void) { return 0; }
static int teardown(void) { return 0; }
```

setup が非 0 を返すとテストは失敗扱いになり、本体は実行されません。

## タグ

`DT_TEST_T` / `DT_TEST_FT` でタグ文字列を付けられます。

```c
DT_TEST_T("uart_smoke", DT_RESET_SOFT, NULL, "uart,smoke", uart_smoke) { ... }
```

ホスト側では `--tag smoke` で対象テストを絞り込みます。
